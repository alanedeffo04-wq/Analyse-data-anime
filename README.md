# Analyse-data-anime
Mon Premier Projet Data : Score Éditorial Anime 
Le contexte
En tant que passionné de données débutant, j'ai voulu aider une plateforme de streaming à faire un choix malin : quels animes mettre en avant ? Comme on n'a pas toujours toutes les infos, j'ai créé un système pour calculer un "Score Éditorial" et trouver les meilleures pépites.
 Ma boîte à outils (Installation)
Pour commencer, j'ai dû préparer mon "atelier" de travail :
• Anaconda : Pour avoir un environnement de code propre.
• PyCharm : Mon éditeur pour écrire et lire mon code.
• Les bibliothèques : J'ai appris à utiliser Pandas (pour manipuler les tableaux), Matplotlib et Seaborn (pour transformer les chiffres en images/graphiques).
 Mon parcours étape par étape
1. L'Audit & Le Nettoyage : C'était mon premier défi. J'ai vérifié les données et j'ai supprimé les doublons. C'est une étape cruciale car des données en double faussent tous les résultats !
2. L'Enrichissement : J'ai ajouté des critères pour rendre les données plus "riches" (par exemple, calculer de nouvelles moyennes).
3. La Corrélation : J'ai cherché à voir si deux choses étaient liées (par exemple : est-ce que les animes les plus longs sont forcément les mieux notés ?).
4. La Visualisation : J'ai transformé mes tableaux de chiffres en graphiques colorés pour que les résultats sautent aux yeux.
 Le Top 10 Ultime
Pour finir, j'ai créé un classement final. Je ne me suis pas arrêté à la simple note générale. J'ai pris en compte :
• La note globale (la popularité).
• Le meilleur épisode (le pic de qualité).
• La régularité (est-ce que c'est bon du début à la fin ?).
Résultat : Une synthèse qui permet de sortir un Top 10 vraiment fiable pour la plateforme.
